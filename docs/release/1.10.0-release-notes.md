# Forge 1.10.0 Release Notes

Date: 2026-03-30
Version: 1.10.0
Status: Public release candidate

## Summary

Forge 1.10.0 turns the repo into a cleaner public-release surface.
The release adds the missing public repo foundation, rewrites the root README for external readers, and clarifies that release readiness is driven by the canonical verification gate instead of host-specific soak requirements.

## Highlights

- `MIT` license is now published at the repo root.
- Root public docs now include contribution, security, and code-of-conduct guidance.
- The root README now explains Forge's product split, release model, and install flow per host.
- Release-facing policy docs now treat extra smoke evidence as optional hardening, not a release precondition.
- Release-hardening tests now protect public-doc hygiene and the current public release policy.

## Install Notes

Codex:

```powershell
python scripts/verify_repo.py
python scripts/build_release.py
python scripts/install_bundle.py forge-codex --activate-codex
```

Antigravity:

```powershell
python scripts/verify_repo.py
python scripts/build_release.py
python scripts/install_bundle.py forge-antigravity --build
```

Runtime tools remain optional and still require explicit `--target` paths.

## Repo Settings

These are manual hosting-platform steps and are not stored in the repo itself:

- enable private vulnerability reporting or security advisories
- confirm default branch protection for `main`
- confirm issue or discussion configuration if public feedback intake is desired

## Publish Order

1. Push the release commit to the canonical branch.
2. Push the release tag.
3. Publish a release entry using these notes plus the top `CHANGELOG.md` entry.
4. Attach or publish `dist/` artifacts if distribution happens through bundled outputs.
5. Announce the install path for `forge-codex`, `forge-antigravity`, and optional runtime tools.

## Known Limits

- Browser-backed live smoke remains optional hardening rather than a mandatory gate.
- Public repo settings still have to be applied on the hosting platform.
