# Forge 1.10.1

Forge 1.10.1 is a public patch release focused on release hygiene, cross-platform CI declaration, and a clearer runtime-state contract for `forge-design`.

## Highlights

- Normalize `CHANGELOG.md` to English-first release prose across the full published history.
- Expand the GitHub Actions verify workflow to a Windows, Ubuntu, and macOS matrix.
- Materialize empty `forge-design-state` layout up front, including `state/renders.jsonl`.
- Add an explicit maintainer contact path in `SECURITY.md` for private follow-up when platform-native reporting is unavailable.
- Add a dedicated regression test for empty `forge-design-state` layout materialization.

## Install

### Codex

```powershell
python scripts/verify_repo.py
python scripts/build_release.py
python scripts/install_bundle.py forge-codex --activate-codex
```

### Antigravity

```powershell
python scripts/verify_repo.py
python scripts/build_release.py
python scripts/install_bundle.py forge-antigravity --build
```

Runtime tools remain optional and still require explicit `--target` paths.

## Verification

## Notes

- Browser-backed live smoke remains optional hardening, not a mandatory release gate.
- The GitHub Actions matrix is configured in the repo; this release body only claims local Windows verification evidence.
- Public host-platform settings such as private vulnerability reporting and security advisories still need to be enabled on the repository host.
