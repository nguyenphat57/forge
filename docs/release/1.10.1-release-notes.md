# Forge 1.10.1 Release Notes

Date: 2026-03-30
Version: 1.10.1
Status: Public patch release candidate

## Summary

Forge 1.10.1 is a follow-up patch release for the public-release snapshot.
It focuses on release hygiene, clearer public-facing docs, stronger CI declaration, and a tighter runtime-state contract for `forge-design`.

## Highlights

- `CHANGELOG.md` is now fully normalized to English-first public prose.
- The GitHub Actions verify workflow now declares a cross-platform matrix for Windows, Ubuntu, and macOS.
- `forge-design` now materializes an empty `forge-design-state` layout up front, including `state/renders.jsonl`.
- `SECURITY.md` now includes an explicit maintainer contact path for private follow-up when platform-native reporting is unavailable.
- A paste-ready GitHub Release body is now available in `docs/release/1.10.1-github-release-body.md`.

## Install Notes

Codex:

```powershell
python scripts/verify_repo.py
python scripts/build_release.py
python scripts/install_bundle.py forge-codex --activate-codex
```

Antigravity:

```powershell
python scripts/verify_repo.py
python scripts/build_release.py
python scripts/install_bundle.py forge-antigravity --build
```

Runtime tools remain optional and still require explicit `--target` paths.

## Verification

Canonical repo verification:

```powershell
python scripts/verify_repo.py
```

Targeted checks run for this patch snapshot:

- `python -m unittest discover packages/forge-design/tests -v`
- `python -m unittest tests.test_release_hardening -v`
- `python scripts/verify_repo.py`

Result for this release candidate:

- `PASS` on 2026-03-30

## Repo Settings

These are manual hosting-platform steps and are not stored in the repo itself:

- enable private vulnerability reporting or security advisories
- confirm default branch protection for `main`
- confirm issue or discussion configuration if public feedback intake is desired

## Publish Order

1. Push the release commit to the canonical branch.
2. Push the release tag.
3. Publish a release entry using `docs/release/1.10.1-github-release-body.md` plus the top `CHANGELOG.md` entry.
4. Attach or publish `dist/` artifacts if distribution happens through bundled outputs.
5. Announce the install path for `forge-codex`, `forge-antigravity`, and optional runtime tools.

## Known Limits

- Browser-backed live smoke remains optional hardening rather than a mandatory gate.
- The cross-platform GitHub Actions matrix is configured in the repo, but this snapshot only has local Windows verification evidence.
- Public repo settings still have to be applied on the hosting platform.
